# Geek精神与GitHub学习笔记
## 1. 我理解的Geek精神
在我看来，**Geek精神**是「用技术折腾世界，以热爱死磕细节」——不是单纯的“技术宅”，而是愿意为了一个有趣的想法/更高效的方案，花时间钻透原理、动手实践，甚至用技术解决生活里的小问题（比如写个脚本自动整理文件）。同时也愿意分享：把自己踩过的坑、写的工具丢到开源社区，让更多人少走弯路。也正如我之前所说的那样无限求知，无限进步，无限热情。


## 2. GitHub是什么？那些术语啥意思？
- **GitHub**：本质是「基于Git的代码托管+全球最大开源社区」——既能存自己的代码、做版本管理，也能抄（不是）参考别人的开源项目，还能和全世界的开发者一起改代码。
- **Pull Request（PR）**：我改了别人项目的代码，想让作者把我的修改合并进去，就提交一个PR，相当于“请你看看我的改动合不合规~”。
- **Fork**：把别人的仓库“复制”一份到我自己的GitHub账号下，我可以随便改（但不影响原仓库），改完想给原作者贡献就提PR。

这些知识是通过**教程+GitHub官方文档+B站的入门视频**学到的~


## 3. 我遇到的GitHub访问问题吗？
我没遇到过“进网站花半天”的情况，不过猜测学长卡住的原因可能是：
- 网络问题：GitHub在国内访问偶尔不稳定，没开加速工具的话加载会很慢；
- 界面陌生：第一次用GitHub，导航栏/功能按钮太多，不知道从哪入手，也可能因为英文大量存在看不懂。


## 4. 这份文档为什么“整洁规范”？
我遵循了这些Markdown规范：
1. **层级清晰**：用`#`/`##`区分标题层级，避免大段无结构文字；
2. **格式统一**：列表用`-`/`1.`、强调用`**加粗**`，风格保持一致；
3. **逻辑分块**：每个问题单独成小节，用空行分隔模块，读起来不拥挤；
4. **术语标注**：关键术语（如PR/Fork）单独说明，降低理解成本。
